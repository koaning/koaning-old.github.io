rock paper starcraft

in this blog post i am going to explore representations of different games through graphs.

rock paper scissors

a simple game of rock paper scissors can be represented through a graph. a node is a choice of a player, an edge is the outcome. if you move your mouse over a node, let's say paper, you will see the nodes that it will beat (rock) and the nodes that will beat it (scissors).

rock paper scissors lizzard spock

this game can be evolved into something more complex, as seen in the big bang theory. but we would still be able to represent this game through a graph.

the interesting thing is that we can see that a game is balanced if the graph is symmetric. if there is no symmetry then a player might be able to exploit a game in a phenomenon.

enter starcraft

personally i feel like starcraft is one of the most complex games out there, so i was wondering if such a game representation would allow me to make the game more comprehendable. i used the official starcraft website as a source of data. for every unit in the game the website lists opponing units that are good or bad against that unit. with this data I was able to create the following graph;

the game obviously involves a lot more than just simply choosing units and the data that i am using is far from enough to state that i have just completely visualised the game. but you can notice the following things;

  • the color of a node in the graph denotes if a unit was either zerg, protoss or terran. notice how the races of starcraft are evenly distributed on the graph
  • notice that the worker units have clustered together
  • if you force one type of unit you become very vunerable unless you are countering an opponent
  • notice how the graph seems to be symmetric

desert strike

again, the game of starcraft requires more skills than just knowledge of the units, so i won't be able to use this to my advantage to a normal game. there are user generated games of starcraft in arcade mode that don't require you to control your units however, so those games might be more hackable with this graph representation. the most popular of these maps is called 'desert strike' so i decided to make a small tool to assist me in these games.

i've added some fancy d3 functionality to it so that you can select what your opponent throws at you to help you determine what units you should be making. in this graph, you can now click the nodes such that you can keep track an opponents army.

i actually got better at desert strike by using this. i didn't win each game, but using this it became much clearer what units i needed to use to counter other units. all despite the fact that the representation is very basic. you could make such a graph more accurate, but you would need more advanced data.
